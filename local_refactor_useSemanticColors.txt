1. Τι είναι αυτό το αρχείο στην πραγματικότητα
Το useSemanticColors.ts δεν είναι ένα απλό hook. Είναι:

🔹 Design-system abstraction layer

🔹 Semantic token resolver

🔹 UI pattern catalog

🔹 Tailwind class factory

🔹 State & interaction contract

Με enterprise ορολογία, αυτή τη στιγμή το αρχείο παίζει 4 ρόλους ταυτόχρονα:

Ρόλος	Enterprise όνομα
Χρώματα	Design Tokens Resolver
Hooks	Presentation Adapters
UI patterns	UI Contracts / Recipes
Tailwind mapping	Implementation Detail
Αυτό δεν είναι λάθος λειτουργικά, αλλά είναι architectural smell σε enterprise κλίμακα.

2. Χρειάζεται διάσπαση;
Ναι, χρειάζεται διάσπαση
Όχι επειδή “είναι μεγάλο”, αλλά επειδή:

❌ Παραβιάζεται το Single Responsibility Principle (SRP) σε design-system επίπεδο
❌ Αναμειγνύονται domain-level semantics με framework implementation
❌ Δεν μπορεί να γίνει independent evolution (π.χ. αλλαγή Tailwind → CSS Modules → Panda CSS)
Σε enterprise περιβάλλον, αυτό θεωρείται God Module.

3. Πώς το βλέπουν αυτό μεγάλες enterprise εταιρείες
Θα σου πω πώς το αντιμετωπίζουν Microsoft, Salesforce, Atlassian, Shopify, Adobe, όχι θεωρητικά.

Κανόνας #1 (απαράβατος)
Semantic meaning ≠ implementation

ΔΕΝ επιτρέπεται:

Επιτρέπεται:

semantic.success = token.success.text
