Αυτό που περιγράφεις είναι κλασική ένδειξη ότι τα κείμενα δεν αντιμετωπίζονται ως “γεωμετρικές οντότητες σε μοντέλο” αλλά ως annotations / screen-space text.
Συνοπτικά: δεν είναι θέμα zoom, είναι θέμα τρόπου render.
Παρακάτω τα πιθανά αίτια, με σειρά πιθανότητας.

1. Τα κείμενα αποδίδονται σε screen space (px) αντί για world/model space
Όταν συμβαίνει αυτό:


❌ το μέγεθος κειμένου δεν αλλάζει με zoom


❌ οι διαστάσεις δεν “δένουν” με τις υπόλοιπες οντότητες


✅ γραμμές, πολυγραμμές, κύκλοι φαίνονται σωστοί


Τεχνικά:


γίνεται κάτι σαν
ctx.font = "14px Arial";

αντί για font που προκύπτει από μονάδες DXF × scale


Αυτό συμβαίνει συχνά όταν:


το text renderer αγνοεί το transform.scale


ή εφαρμόζει annotation scaling χωρίς να το θες



2. Χάνεται το πραγματικό text height του DXF και πέφτει σε default
Στο DXF:


TEXT / MTEXT έχουν ύψος (group code 40) σε μονάδες σχεδίου


Αν:


δεν διαβαστεί σωστά το 40


ή χαθεί καθ’ οδόν (parse → entity → render)


τότε:


όλα τα κείμενα καταλήγουν με ίδιο default μέγεθος


και φαίνονται λάθος σε σχέση με την κάτοψη


Σύμπτωμα:


όλα τα texts “μοιάζουν” μεταξύ τους, ανεξαρτήτως τι γράφουν



3. Το DXF είναι σε διαφορετικές μονάδες (INSUNITS) και το text όχι
Π.χ.:


σχέδιο σε μέτρα


text height γραμμένο σαν χιλιοστά ή αντίστροφα


Αν:


εφαρμόζεις unit scaling στις γεωμετρίες


αλλά όχι στο text height


τότε:


γραμμές σωστές


γράμματα λάθος


(πολύ συχνό σε AutoCAD → custom viewers)

4. MTEXT vs TEXT δεν αντιμετωπίζονται το ίδιο
Πολλά viewers:


χειρίζονται σωστά TEXT


αλλά το MTEXT:


έχει πλάτος, line height, attachment point


συχνά “πέφτει” σε screen-space fallback




Αν το πρόβλημα εμφανίζεται κυρίως σε:


ονόματα χώρων


πολυγραμμικά κείμενα
→ είναι ένδειξη MTEXT handling bug



5. Εσκεμμένο “annotation mode”
Σε αρκετά συστήματα υπάρχει επιλογή τύπου:


Keep text readable


Annotation scaling


Fixed screen size text


Αν αυτό είναι:


ενεργό, τότε το zoom δεν επηρεάζει το μέγεθος



Πώς το επιβεβαιώνεις άμεσα (χωρίς κώδικα)
Δες αν ισχύει κάποιο από τα παρακάτω:


Όλα τα κείμενα έχουν περίπου ίδιο ύψος, άσχετα με το σχέδιο


Κάνεις zoom ×10 και:


γραμμές μεγαλώνουν


γράμματα μένουν ίδια




Αν αλλάξεις canvas size / DPR, αλλάζει το text αλλά όχι οι γραμμές


Αν ναι → screen-space rendering.

Bottom line
Το πρόβλημα δεν είναι στο zoom, ούτε στο DXF αρχείο κατ’ ανάγκη.
Είναι ένα από τα εξής:


text rendered σε pixels αντί για world units


χαμένο ή αγνοημένο DXF text height (group code 40)


annotation scaling ενεργό


ασυνεπές unit scaling μόνο για text


Αν θέλεις, στο επόμενο βήμα:


σου λέω ακριβώς τι να ελέγξεις στον renderer


ή σου δίνω 2-3 συγκεκριμένα console checks για να αποδείξεις ποιο από τα παραπάνω συμβαίνει.
