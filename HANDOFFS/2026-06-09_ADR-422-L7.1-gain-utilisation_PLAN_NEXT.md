## 1) ΤΙ ΘΑ ΚΑΝΕΙΣ (L7.1)

Το L7 δίνει τη **μεικτή** ζήτηση (gross losses) αμελώντας τα ωφέλιμα θερμικά κέρδη → **υπερεκτιμά**. Οι «μεγάλοι παίχτες» (Revit Energy, 4M-FineHEAT-KENAK) υπολογίζουν **καθαρή** ζήτηση αφαιρώντας τα **αξιοποιήσιμα** κέρδη (εσωτερικά + ηλιακά) μέσω **συντελεστή αξιοποίησης `η_gn`** (EN ISO 13790 simplified seasonal).

**Στόχος L7.1:** ανά χώρο, καθαρή ετήσια ζήτηση
```
Q_H,net = max(0, Q_loss − η_gn · (Q_int + Q_sol))
```
όπου `Q_loss` = η μεικτή του L7, `Q_int` = εσωτερικά κέρδη (ανά χρήση), `Q_sol` = ηλιακά κέρδη (από υαλοπίνακες), `η_gn` = συντελεστής αξιοποίησης κερδών (συνάρτηση του λόγου κερδών/απωλειών `γ`). Η ειδική ζήτηση `q_H` και η **κατηγορία** βασίζονται πλέον στην **καθαρή** ζήτηση· η μεικτή παραμένει ορατή (breakdown, όπως Revit).

**Όλα derived** (μηδέν persist), **advisory**, **section-only** στο υπάρχον report → **μηδέν shared αρχείο με boiler agent**.

## 2) ΑΠΟΦΑΣΕΙΣ (Revit-grade, ΗΔΗ LOCKED — μην ξανα-ρωτήσεις)

- **D-A — μέθοδος = EN ISO 13790 simplified seasonal gain utilisation.** Πηγή απωλειών = τα ήδη-υπολογισμένα L7/L1 (μηδέν re-resolve geometry). Καθαρά αριθμητική.
- **D-B — νέο config** `bim/thermal/heat-load/annual-gains-config.ts` (ΟΧΙ fork· ξεχωριστό SRP από το `annual-energy-config.ts`, mirror της isolation D-B των L6/L7). Περιέχει (config/types — εξαιρείται 500-line):
  - `INTERNAL_GAIN_W_PER_M2: Record<ThermalSpaceUseType, number>` — εσωτερικά κέρδη ανά χρήση (ΤΟΤΕΕ 20701-1, documented editable· π.χ. κατοικία/υπνοδωμάτιο ~4, σαλόνι/κουζίνα ~6, γραφείο ~varies). **Reuse `ThermalSpaceUseType`.**
  - `HEATING_SEASON_HOURS: Record<ClimateZone, number>` — ώρες περιόδου θέρμανσης ανά ζώνη (documented· ψυχρότερη ζώνη → περισσότερες ώρες· συνεπές με τα HDD του L7).
  - `SEASONAL_SOLAR_IRRADIATION_KWHM2: Record<ClimateZone, number>` — εποχιακή ηλιακή ακτινοβολία σε **κατακόρυφη** επιφάνεια (kWh/m²·περίοδο), **orientation-agnostic μέση τιμή v1** (documented· θερμότερη/νότια ζώνη → μεγαλύτερη). Ο προσανατολισμός = future L7.2.
  - σταθερές `GLAZING_SOLAR_FACTOR_G` (g-value, ~0.6 διπλός υαλοπίνακας), `FRAME_FACTOR` (~0.7), `SHADING_FACTOR` (~0.9) — documented editable.
  - `UTILISATION_NUMERIC_PARAM` `a0` (=1.0, simplified seasonal — documented).
  - Getters: `getInternalGainWperM2(use)`, `getHeatingSeasonHours(zone)`, `getSeasonalSolarIrradiation(zone)` + **pure** `computeGainUtilisation(gainLossRatio: number): number`.
- **D-C — `computeGainUtilisation(γ)`** (EN ISO 13790 §12.2.1.1, simplified):
  - `γ ≤ 0` ⇒ `η = 1` (κανένα κέρδος ή μηδενικές απώλειες → όλα αξιοποιήσιμα / no-op).
  - `γ = 1` ⇒ `η = a0 / (a0 + 1)`.
  - αλλιώς ⇒ `η = (1 − γ^a0) / (1 − γ^(a0+1))`.
  - **clamp `η ∈ [0, 1]`.** Pure, full unit-testable.
- **D-D — επέκταση του pure aggregator** `derive-annual-energy.ts` (ΟΧΙ νέο αρχείο· additive στο υπάρχον):
  - Per-row πρόσθεσε: `grossDemandKWh` (= το σημερινό `Q_loss`), `internalGainKWh`, `solarGainKWh`, `utilisation`, `netDemandKWh`. **Το headline `annualDemandKWh` γίνεται = `netDemandKWh`** (ώστε downstream class/KPIs = καθαρή). Κράτα `grossDemandKWh` ορατό.
  - `Q_int = getInternalGainWperM2(space.params.useType) · A_floor · getHeatingSeasonHours(zone) / 1000` [kWh].
  - `Q_sol = Σ_windows (A_window · g · F_frame · F_shading · I_season(zone))`. **Window area = από τα ήδη-resolved `result.boundaries`** όπου `kind==='window' && condition==='external-air'` (μηδέν re-resolve). g/F από config.
  - `γ = (Q_int + Q_sol) / Q_loss` (guard `Q_loss>0`· αλλιώς γ→0 ⇒ η=1, net=max(0, −...)=0).
  - σύνοψη: `totalNetKWh`, `totalGrossKWh`, `totalInternalGainKWh`, `totalSolarGainKWh`, ειδική **καθαρή** `q_net = ΣQ_net/ΣA → energyClass`.
- **D-E — UI = επέκταση 7ης section + summary** (ΟΧΙ νέα section, ΟΧΙ νέο widget). Στον πίνακα L7 πρόσθεσε στήλες «Μεικτή (kWh)», «Κέρδη (kWh)», «Αξιοπ. (%)», «Καθαρή (kWh)» (η υπάρχουσα `annualDemand` → καθαρή· πρόσθεσε gross/gains/utilisation). Στη σύνοψη το `annualEnergy` KPI = καθαρή· πρόσθεσε προαιρετικό «μεικτή» KPI αν χωράει. `climateZone: null` ⇒ 0 γραμμές (graceful, ως τώρα).
- **D-F — μονάδες**: kWh `count` (ακέραιο)· utilisation = ποσοστό `number` (×100, 2-δεκ) **ή** κράτα 0-1 με `number` — διάλεξε ποσοστό για readability (label «Αξιοπ. (%)»). **Καμία νέα `ScheduleColumnValueType`** (FULL SSOT· `number`=2-δεκ, `count`=ακέραιο).
- **D-G — κατηγορία = καθαρή ζήτηση** (πιο ρεαλιστική)· documented στο ADR. Τα bands του `annual-energy-config.ts` ΜΕΝΟΥΝ ίδια (indicative).

### ΦΥΣΙΚΗ (κρίσιμη — μην την αλλάξεις)
```
Q_loss   = H · HDD · 24 / 1000                          [kWh]  (L7, gross — αμετάβλητο)
Q_int    = q_int(use) · A_floor · hours_season / 1000   [kWh]
Q_sol    = Σ_win A_win · g · F_frame · F_shading · I_season(zone)  [kWh]
γ        = (Q_int + Q_sol) / Q_loss
η_gn     = (1 − γ^a0)/(1 − γ^(a0+1))    (γ≠1)·  a0/(a0+1) (γ=1)·  1 (γ≤0)·  clamp[0,1]
Q_H,net  = max(0, Q_loss − η_gn · (Q_int + Q_sol))
q_net    = ΣQ_H,net / ΣA  →  κατηγορία (bands L7)
```
**ΣΗΜ:** το `H` (συντ. απωλειών) προκύπτει ήδη στο L7 (`(transmissionW+ventilationW)/deltaTC`, reheat εξαιρείται). Το `Q_loss` per-space το ΞΑΝΑΫΠΟΛΟΓΙΖΕΙΣ ήδη μέσα στο `deriveAnnualHeating` — απλώς κράτα το ως `grossDemandKWh` και πρόσθεσε τα κέρδη/net.
