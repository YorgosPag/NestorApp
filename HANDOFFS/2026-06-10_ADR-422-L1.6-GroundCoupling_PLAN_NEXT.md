**Το πρόβλημα που λύνει το L1.6:** το δάπεδο σε επαφή με το έδαφος (`condition: 'ground'`) χρησιμοποιεί σήμερα **σταθερό μειωτικό συντελεστή `b=0.5`** (`heat-load-config.ts:33` `BOUNDARY_TEMPERATURE_FACTOR.ground`). Αυτό αγνοεί ότι η πραγματική απώλεια προς το έδαφος εξαρτάται από τη **γεωμετρία** (λόγος εμβαδού/περιμέτρου — edge effect: μικρά/στενόμακρα δάπεδα χάνουν πολύ περισσότερο ανά m²) και την **αντίσταση του εδάφους**. Οι «μεγάλοι παίχτες» (Revit Energy «Ground», 4M-FineHEAT, ΚΕΝΑΚ/ΤΟΤΕΕ 20701-2) το υπολογίζουν με **EN ISO 13370**: χαρακτηριστική διάσταση `B′ = A / (½·P_exposed)` + ισοδύναμο πάχος `d_t` (αντίσταση slab + εδάφους) → ακριβές `U_g` (floor-to-external, ενσωματώνει τη διαδρομή εδάφους → χρησιμοποιείται με πλήρες ΔΤ, b=1.0). Είναι το **μεγαλύτερο εναπομείναν φυσικό κενό** του μοντέλου και βελτιώνει τη **βάση** (L1 `Φ`) που τροφοδοτεί ΟΛΑ τα downstream (L2-L7).

## 1) ΤΙ ΘΑ ΚΑΝΕΙΣ (L1.6)

Αντικατέστησε το flat `b=0.5` του δαπέδου επί εδάφους με **EN ISO 13370 ισοδύναμη διαπερατότητα** ανά ground floor boundary:
```
Για κάθε δάπεδο επί εδάφους (kind='floor', condition='ground'):
  B′  = A_floor / (0.5 · P_exposed)                          [χαρακτηριστική διάσταση, m]
  d_t = w + λ_g · (R_si + R_f + R_se)                        [ισοδύναμο πάχος, m]
        όπου R_f = 1 / U_floor_construction (η αντίσταση του ίδιου του slab)
  Αν d_t < B′ (αμόνωτο/μέτρια μονωμένο):
     U_g = (2·λ_g / (π·B′ + d_t)) · ln(π·B′/d_t + 1)
  Αν d_t ≥ B′ (καλά μονωμένο):
     U_g = λ_g / (0.457·B′ + d_t)
απώλεια δαπέδου = U_g · A · 1.0 · ΔΤ   (το U_g ΕΝΣΩΜΑΤΩΝΕΙ τη διαδρομή εδάφους → b=1)
```
**Πηγές δεδομένων (ΟΛΕΣ υπάρχουν):** `A_floor` = `space.geometry.area` (περνά ήδη στο `resolveFloorBoundary`)· `P_exposed` = άθροισμα μηκών των **εξωτ.** τοίχων που οριοθετούν τον χώρο (ο resolver ξέρει ήδη `ctx.exteriorWallIds` + `matchedLenScene·sceneToM` ανά τοίχο στο `resolveWallBoundaries` — accumulate & πέρασέ το στο `resolveFloorBoundary`)· `R_f` από το `DEFAULT_FLOOR_U_WPER_M2K` (ή το πραγματικό slab U)· `w` = αντιπροσωπευτικό πάχος εξωτ. τοίχου (config default ή μέσος όρος).

**ΣΚΟΠΙΜΩΣ ΕΚΤΟΣ v1 (= future):** ground **walls** (υπόγειο — σήμερα οι τοίχοι ΔΕΝ παίρνουν ποτέ `condition:'ground'`, μόνο δάπεδα → bounded scope)· βάθος υπογείου `z`· περιμετρική μόνωση edge insulation (ψ_g linear)· εποχιακή διακύμανση θερμοκρασίας εδάφους (annual periodic 13370 §E)· per-soil-type λ catalog (config const v1).

## 2) ΑΠΟΦΑΣΕΙΣ (Revit-grade, στρατηγικά LOCKED — οριστικοποίησε mechanics σε Plan Mode)

- **D-A — EN ISO 13370 simplified steady-state slab-on-ground (§9.1/§9.3), ΟΧΙ flat b.** Η χαρακτηριστική διάσταση `B′` + ισοδύναμο πάχος `d_t` δίνουν το `U_g` που ΕΝΣΩΜΑΤΩΝΕΙ τη διαδρομή εδάφους → εφαρμόζεται με **b=1.0** (πλήρες ΔΤ). **ΓΙΑΤΙ:** Revit/4M/ΚΕΝΑΚ· το flat 0.5 είναι geometry-blind (edge effect αγνοημένο).
- **D-B — `U_g` two-branch (§9.3):** `d_t<B′` → `(2λ_g/(π·B′+d_t))·ln(π·B′/d_t+1)`· `d_t≥B′` → `λ_g/(0.457·B′+d_t)`. **NEW pure SSoT** `ground-coupling.ts` (`computeCharacteristicDimension`, `computeEquivalentThickness`, `computeGroundUValue`). Καθαρή αριθμητική, full unit-testable.
- **D-C — config σταθερές (NEW στο `heat-load-config.ts`):** `SOIL_THERMAL_CONDUCTIVITY_WPER_MK ≈ 2.0` (λ_g, ΤΟΤΕΕ 20701-2 τυπικό έδαφος)· `GROUND_FLOOR_INTERNAL_RESISTANCE_R_SI ≈ 0.17` (R_si δαπέδου, EN ISO 6946 ροή προς τα κάτω)· `EXTERNAL_SURFACE_RESISTANCE_R_SE` (reuse 0.04 από `annual-gains-config` ή τοπικό)· `DEFAULT_GROUND_WALL_THICKNESS_M ≈ 0.30` (w αντιπροσωπευτικό· ή μέσος εξωτ. τοίχων). Editable defaults, documented.
- **D-D — exposed perimeter `P` = Σ μηκών εξωτ. τοίχων του χώρου** (όχι το γεωμετρικό perimeter — μόνο οι όψεις προς εξωτ./έδαφος μετράνε στο edge loss). Accumulate στο `resolveWallBoundaries` loop (έχει `isExterior` + `matchedLenScene`)· πέρασέ το στο `resolveFloorBoundary`. **Fallback** αν `P≤0` → flat `b=0.5` (zero-regression για degenerate γεωμετρίες).
- **D-E — ⚠️ ΟΧΙ zero-regression (cascade σε όλα τα downstream).** Αλλάζει το `Φ` ΟΛΩΝ των χώρων με δάπεδο επί εδάφους → αλλάζει `transmissionW`/`totalW`/`specificLoadWperM2` (L1) → radiator/pipe sizing (L2/L3) → balancing (L4) → report (L5) → **ζήτηση (L7)**. **Update baselines** σε ΠΟΛΛΑ thermal suites (όχι μόνο 1-2). Αυτό είναι το κύριο ρίσκο — δες §4/§5.
- **D-F — ενσωμάτωση στον engine: ΛΥΣΕ ΣΤΟ PLAN MODE (§5).** Δύο καθαρές επιλογές, διάλεξε ΜΙΑ (σύσταση: η 1η):
  1. **Stamp effective `U_g` + per-boundary temperature-factor override:** ο resolver υπολογίζει το `U_g`, το βάζει ως `boundary.uValue` ΚΑΙ στάμπα `groundTemperatureFactor?: 1.0`· ο engine: `factor = boundary.groundTemperatureFactor ?? getBoundaryTemperatureFactor(condition)`. Absent → flat 0.5 (zero-regression by construction για μη-ground/legacy). **Καθαρό mirror του L7.x per-boundary stamping** + κρατά το ΔU_TB θερμογέφυρας να λειτουργεί σωστά (το `boundaryReceivesThermalBridge('floor','ground')` ΔΕΝ είναι σήμερα true — επιβεβαίωσε: `THERMAL_BRIDGE_KINDS` έχει 'floor' αλλά εφαρμόζεται μόνο σε ground/external → θα προσαυξήσει· πρόσεξε διπλομέτρηση edge).
  2. **Stamp effective `b_g = U_g / U_floor`** ως override, κράτα `uValue = U_floor`. Μαθηματικά ισοδύναμο (`U_floor·b_g = U_g`) αλλά λιγότερο διαφανές (το `b` παύει να είναι «temperature factor»). **ΟΧΙ προτιμώμενο.**
- **D-G — ΚΑΝΕΝΑ νέο UI control / i18n string / report column** (auto-computed από geometry, mirror θερμογεφυρών/13790). Ξεχωριστή στήλη «Απώλεια εδάφους» = future (i18n shared με boiler dev).

### ΦΥΣΙΚΗ (κρίσιμη)
```
B′ = A / (0.5·P)              [m, χαρακτηριστική διάσταση — μεγάλο δάπεδο→μεγάλο B′→μικρό U_g]
d_t = w + λ_g·(R_si+R_f+R_se) [m, ισοδύναμο πάχος — καλή μόνωση→μεγάλο d_t→μικρό U_g]
U_g = two-branch (πάνω)       [W/m²K, ΕΝΣΩΜΑΤΩΝΕΙ έδαφος → εφαρμόζεται με b=1]
Φ_floor = U_g · A · ΔΤ
```
**ΣΗΜ:** `R_se`/`getHeatingSeasonHours` (αν χρειαστεί) reuse· `space.geometry.area` reuse· `ctx.exteriorWallIds`/`matchedLenScene` reuse. ΜΟΝΟ το `ground-coupling.ts` + οι 3-4 config σταθερές + το perimeter accumulation + το engine override είναι νέα.

## 5) ⚠️ ΚΡΙΣΙΜΑ ΘΕΜΑΤΑ (ΛΥΣΕ ΤΑ ΣΤΟ PLAN MODE — μην τα προσπεράσεις)

1. **Engine integration mechanics (D-F):** διάλεξε option 1 (effective U + factor override) vs option 2 (b_g override). **Σύσταση: option 1.** ⚠️ Πρόσεξε το **ΔU_TB θερμογέφυρας**: σήμερα `boundaryReceivesThermalBridge('floor','ground')=true` → προστίθεται `ΔU_TB` στο floor. Με το νέο `U_g` (που ήδη ενσωματώνει edge loss), αποφάσισε αν το ΔU_TB διπλομετρά το perimeter edge → ίσως εξαιρέσεις το ground-floor από το TB surcharge (το 13370 `U_g` ΗΔΗ περιλαμβάνει τον edge). Τεκμηρίωσε.
2. **ΒΑΘΜΟΝΟΜΗΣΗ (worked example, ΠΡΙΝ έγκριση):** δωμάτιο 4×4 (A=16), 4 εξωτ. όψεις → P=16 → B′=16/(0.5·16)=**2.0 m**· w=0.30, λ_g=2.0, R_si=0.17, R_se=0.04, slab U=0.5→R_f=2.0 → d_t=0.30+2.0·(0.17+2.0+0.04)=**4.72 m**· d_t≥B′ → `U_g=2.0/(0.457·2.0+4.72)=2.0/5.634=`**0.355 W/m²K** (effective b_g=0.355/0.5=**0.71**, vs σημερινό flat 0.5·0.5=0.25 → **+42%** edge-dominated, σωστό για μικρό συμπαγές δωμάτιο)· μεγάλο 20×20 (A=400, P=80) → B′=10, d_t=4.72<10 → branch-1 `U_g=(4.0/(π·10+4.72))·ln(π·10/4.72+1)=0.1107·2.036=`**0.225** (vs flat 0.25 → ελαφρώς λιγότερο). **Δείξε ότι μικρό δάπεδο→περισσότερη απώλεια / μεγάλο→λιγότερη (edge effect)** ΠΡΙΝ εγκρίνει ο Giorgio. ⚠️ Ξανα-υπολόγισε με τις ΤΕΛΙΚΕΣ σταθερές που θα κλειδώσεις (το L7.8-B handoff είχε αριθμητικό λάθος — μην το επαναλάβεις, παρήγαγε κάθε τιμή από τον τύπο).
