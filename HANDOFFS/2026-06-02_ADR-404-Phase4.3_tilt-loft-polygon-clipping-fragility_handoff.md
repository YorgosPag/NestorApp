## 4. ΣΥΣΤΗΜΕΝΗ ΛΥΣΗ (διάλεξε/συνδύασε — ο στόχος = robustness)

**Α. Robust analytic critical-t (προτιμητέο — μηδέν clipper στο detection).**
Το αρχικό αναλυτικό (vertex-on-edge: host vertex σε quad edge + quad vertex σε host edge) **έχανε** το t≈0.01 split γιατί ήταν **edge-edge bridge** (το δοκάρι γεφυρώνει — οι ΔΥΟ μακριές ακμές host τέμνουν την ίδια διατομή). Πρόσθεσε **τρίτο** κριτήριο: critical t όταν δύο host edges τέμνουν την ΙΔΙΑ quad edge ταυτόχρονα (bridge) ή όταν ο αριθμός crossings host-boundary × quad-boundary αλλάζει. Έτσι O(few) candidates, **κανένα** `safeDifference`.

**Β. Αν χρειαστεί boolean: σκλήρυνε / απόφυγε το fragile clipper.**
- (i) Καθάρισε input πριν το clip (dedupe near-collinear vertices, snap σε grid)· δοκίμασε διαφορετικό `scale` στο `runScaled` όταν αποτυγχάνει (retry-with-jitter στις coords ~1e-7).
- (ii) Όταν `safeDifference` επιστρέφει κενό ΕΝΩ προφανώς υπάρχει επικάλυψη → **retry** με μικρό jitter στο t (±1e-4)· τα critical-t θέλουμε *κοντά* όχι *πάνω* στο degenerate.
- (iii) Εναλλακτική βιβλιοθήκη/μέθοδος: Sutherland-Hodgman line-clip κατά τις **κατακόρυφες παρειές του δοκαριού** (γραμμές) — robust, χωρίς ring-completion failures (το δοκάρι κόβει με επίπεδα → quad ∩ halfplane). quad−beam = quad clipped έξω από κάθε beam edge· για convex beam δούλεψε per-edge.

**Γ. Performance (live preview):** το detection ΔΕΝ πρέπει να τρέχει 48× ανά frame. Με αναλυτικό (Α) λύνεται. Αλλιώς cache/throttle στο preview.

**Δ. Degenerate-at-Hu:** αφού το split είναι στο t≈0, σκέψου να αντιμετωπίσεις την περίπτωση «το δοκάρι γεφυρώνει στο Hu» ρητά (η outside είναι ήδη 2 πολύγωνα στο Hu → το `safeDifference(quad, host@0)` που χρησιμοποιεί ΚΑΙ ο lower-prism loop πρέπει να είναι robust εκεί).
