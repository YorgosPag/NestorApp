## 1) ΤΙ ΘΑ ΚΑΝΕΙΣ (L1.5)

Σήμερα το engine υπολογίζει **μόνο 2 από τους 3 όρους** του EN 12831 / ΤΟΤΕΕ 20701-1 design heat load:

```
ΣΗΜΕΡΑ:   Φ = Φ_T + Φ_V
EN 12831:  Φ_HL = Φ_T,i + Φ_V,i + Φ_RH,i           (+ θερμογέφυρες στο Φ_T)
```

Λείπουν δύο πράγματα που οι «μεγάλοι παίχτες» (Revit Heating Loads / 4M-FineHEAT) έχουν:

1. **Θερμογέφυρες (Thermal Bridges, ΔU_TB)** — προσαύξηση του U των αδιαφανών εξωτερικών στοιχείων (απλοποιημένη μέθοδος EN 12831-1 §6.3.2: `U_corr = U + ΔU_TB`). Τυπικές τιμές `ΔU_TB` ≈ 0.05 (χαμηλές) / 0.10 (μέτριες) / 0.15 (υψηλές) W/m²K. Refines το `Φ_T`.
2. **Επανέναρξη / Reheat (Φ_RH)** — επιπλέον ισχύς για διακοπτόμενη λειτουργία (νυχτερινό setback): `Φ_RH = A_floor · f_RH`, όπου `f_RH` (W/m²) από πίνακα EN 12831 (συνεχής λειτουργία ⇒ `f_RH = 0`).

**Αποτέλεσμα:** πιο ακριβές, κανονιστικά πλήρες φορτίο → **όλα τα downstream (L2 σώματα, L3 σωλήνες, L4 balancing, L5 report) γίνονται σωστότερα ΔΩΡΕΑΝ** (re-derive). Όλα **derived**, μηδέν persist στους υπολογισμούς.

**ΠΡΩΤΟ ΒΗΜΑ: Plan Mode** — διάβασε τον engine + config + resolver + types, πάρε τις αποφάσεις (§2), ζήτα έγκριση.

## 2) ΑΠΟΦΑΣΕΙΣ ΝΑ ΚΛΕΙΔΩΣΕΙΣ ΣΤΟ PLAN MODE (πρότεινε + δικαιολόγησε)

- **D-A (μέθοδος θερμογεφυρών):** **Πρότεινε την απλοποιημένη blanket μέθοδο EN 12831-1** (`U_corr = U + ΔU_TB` ανά αδιαφανές εξωτ. στοιχείο), ΟΧΙ τη λεπτομερή `Σ Ψ·l` (απαιτεί μήκη ακμών/junctions = γεωμετρία-βαριά, future). Δικαιολόγησε (ίδια επιλογή με Revit «thermal bridge» surcharge· SSoT-clean· μηδέν νέα γεωμετρία). **Σε ποια στοιχεία;** πρότεινε `external-air` (και προαιρετικά `ground`) — ΟΧΙ `adjacent-heated` (b=0). Παράθυρα/πόρτες: το ΔU_TB εφαρμόζεται συνήθως στα **αδιαφανή** (το frame TB είναι ήδη στο U_w)· πάρε τη Revit-grade απόφαση (πρότεινε opaque-only: wall/floor/roof/ceiling).
- **D-B (μοντέλο reheat):** `Φ_RH = A_floor · f_RH`. Πρότεινε `f_RH` από **config preset** ανά «λειτουργία θέρμανσης» (συνεχής=0 / διακοπτόμενη ελαφριά / διακοπτόμενη βαριά) — default **συνεχής (0)** ώστε zero-regression για τα υπάρχοντα. Πηγή `A_floor`: ήδη υπάρχει `floorArea` στο `SpaceHeatLoadInput` (το ίδιο που δίνει το W/m²).
- **D-C (πηγή τιμών — building default vs per-space override):** **Πρότεινε v1 = building/use-derived defaults ΧΩΡΙΣ νέα persisted params** (μηδέν schema/command/rules surface → πλήρης isolation από τον boiler agent), με **per-space override ως documented Slice B** (mirror ΑΚΡΙΒΩΣ του υπάρχοντος `setpointTempC?`/`airChangesPerHour?` override pattern στο `ThermalSpaceParams` + tab). Δικαιολόγησε: ξεκινάς από καθαρό, cascading-correct engine χωρίς να αγγίξεις persistence/ribbon.
- **D-D (breakdown στο result):** **Πρότεινε** `SpaceHeatLoadResult += reheatW` (+ προαιρετικά `thermalBridgeW` για το breakdown), ΚΑΙ το `transmissionW` να **περιλαμβάνει** την προσαύξηση TB (ώστε `totalW = transmissionW + ventilationW + reheatW`). **Additive πεδία** → μηδέν break στους consumers (overlay/readout/L5 report διαβάζουν `totalW`/`transmissionW`/`ventilationW`).
- **D-E (επιφάνεια UI):** v1 = **μηδέν νέο ribbon control** (οι defaults τρέχουν αυτόματα· το αποτέλεσμα φαίνεται στο overlay/readout/L5 ήδη). Προαιρετικό micro-add: στήλη «Επανέναρξη (W)» / «Θερμογέφυρες (W)» στον πίνακα φορτίων του **L5 report** (δικό σου αρχείο `bim/thermal/report/thermal-study-report.ts` — same isolation domain, ΟΧΙ shared). Per-space override controls = Slice B.

## 3) SSoT ΘΕΜΕΛΙΟ — REUSE, ΜΗΝ FORK (επιβεβαιωμένο code)

**Heat-load engine (εκεί γίνεται η δουλειά — ΟΛΑ pure):**
- `bim/thermal/heat-load/heat-load-engine.ts` → `computeSpaceHeatLoad(input)`. Σήμερα: `transmissionW = Σ U·A·b·ΔΤ` (μέσω `computeBoundaryLoss`) + `ventilationW = 0.34·n·V·ΔΤ`. **Εδώ μπαίνει**: (α) `U_corr = U + ΔU_TB` στο `computeBoundaryLoss` (μόνο για τις σχετικές conditions), (β) νέος `computeReheat(floorArea, f_RH)` + `totalW += reheatW`.
- `bim/thermal/heat-load/heat-load-types.ts` → `SpaceHeatLoadInput` (πρόσθεσε `thermalBridgeSurchargeWperM2K?` + `reheatFactorWperM2?`) + `SpaceHeatLoadResult` (πρόσθεσε `reheatW` + προαιρ. `thermalBridgeW`) + `BoundaryHeatLoss` (αν θες να εκθέσεις το TB ανά στοιχείο).
- `bim/thermal/heat-load/heat-load-config.ts` → **config-only αρχείο (no size limit)**: πρόσθεσε `THERMAL_BRIDGE_SURCHARGE_PRESETS` (none/low/medium/high → ΔU_TB) + `REHEAT_FACTOR_PRESETS` (συνεχής/διακοπτόμενη → f_RH) + resolvers (mirror του `getBoundaryTemperatureFactor`). **ΟΛΕΣ οι τιμές εδώ — ΠΟΤΕ inline literal στο engine** (όπως ήδη το `b`-factor + `AIR_VENTILATION_FACTOR=0.34`).
- `bim/thermal/heat-load/space-boundary-resolver.ts` + `hooks/data/useHeatLoadInputs.ts` → εδώ **resolve-άρεις** το ΔU_TB + f_RH (building/use default) και τα περνάς στο `SpaceHeatLoadInput`. (Ο resolver είναι ο μόνος που ξέρει scene/building· το engine μένει καθαρή αριθμητική.)
- `bim/thermal/heat-load/derive-space-heat-loads.ts` → aggregation ορόφου (min/max/totalW) — απλώς θα δει το νέο `totalW`. Πιθανόν μηδέν αλλαγή.

**Πρότυπα (mirror, ΜΗΝ fork):**
- **Config + resolver pattern:** `BOUNDARY_TEMPERATURE_FACTOR` + `getBoundaryTemperatureFactor` (exhaustive Record + getter) στο `heat-load-config.ts` — κλωνοποίησέ το για TB/reheat presets.
- **Per-space override pattern (αν κάνεις Slice B):** `ThermalSpaceParams.setpointTempC?` / `airChangesPerHour?` (optional override, default από `thermal-space-use-catalog`) — ΑΚΡΙΒΩΣ αυτό το «type default, instance override» μοτίβο για `thermalBridgeLevel?` / `reheatMode?`. Tab controls: `contextual-thermal-space-tab` + bridge + command-keys (όπως setpoint/ACH).
- **Worked-example tests:** `bim/thermal/heat-load/__tests__/` (engine 11 + derive 6) — πρόσθεσε TB + reheat worked rows (ΤΟΤΕΕ/EN 12831 αριθμοί) + zero-regression (default ⇒ ίδιο φορτίο με σήμερα).

**Read-model μονάδες (μην τις μπερδέψεις):** U **W/m²K**· ΔU_TB **W/m²K**· A/floorArea **m²**· f_RH **W/m²**· Φ/φορτία **W**· ΔΤ **°C/K**.

## 5) ΜΟΝΑΔΕΣ + SSoT ΠΑΓΙΔΕΣ

- **ΔU_TB = W/m²K** (προστίθεται στο U πριν το `·A·b·ΔΤ`) — **ΟΧΙ** W. **f_RH = W/m²** (πολλαπλασιάζεται με `A_floor` σε m²) → W.
- **b-factor + ΔU_TB:** πρόσεξε τη σειρά — `U_corr·A·b·ΔΤ` (το ΔU_TB μπαίνει στο U, το b μένει). Για `adjacent-heated` (b=0) το TB δεν έχει νόημα (μηδέν απώλεια) — gate σε `external-air` (±`ground`).
- **Zero-regression είναι ΚΡΙΣΙΜΟ:** default ΔU_TB=0 + f_RH=0 ⇒ **ακριβώς** το σημερινό φορτίο. Γράψε test που το επιβεβαιώνει (υπάρχοντα 11 engine tests πρέπει να μείνουν πράσινα ΧΩΡΙΣ αλλαγή τιμών).
- **ΟΛΕΣ οι τιμές στο config** (`heat-load-config.ts`) — ΠΟΤΕ inline literal στο engine (κανόνας SSoT, όπως το `0.34`/`b`).
- **i18n SSoT (αν αγγίξεις report/tab):** keys ΠΡΩΤΑ σε `el` **ΚΑΙ** `en`· numeric+unit («W»/«W/m²K») επιτρεπτά· τα report labels ζουν στο `dxf-viewer-shell` namespace (`thermalStudyReport.*`).
- **ΜΗΝ persist-άρεις** τους υπολογισμούς — pure derive (mirror L1-L5). (Τα per-space override params του Slice B ΕΙΝΑΙ persisted — αυτό είναι ΟΚ, είναι input όχι output.)
