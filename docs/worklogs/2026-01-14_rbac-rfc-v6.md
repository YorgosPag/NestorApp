## Step 1 — P0-1 Fix: Owners/Grants Parent Validation

**Files**:
- `docs/rfc/authorization-rbac.md` (lines 819-827, 843-852)

## Step 2 — P0-2 Fix: Audit Logs Tenant Isolation

**Change**: Added companyId to audit_logs schema + tenant-bound read rule

**Files**:
- `docs/rfc/authorization-rbac.md` (lines 164-174 schema, lines 892-897 rules)

**Rationale**:
- ChatGPT identified cross-tenant leak: `isCompanyAdmin()` allowed any company_admin to read ALL audit logs
- Fix uses `isCompanyAdminOfCompany(resource.data.companyId)` for tenant isolation

**Verification**:
- Manual review confirms only same-tenant admins can read logs
- Self-read (actorId == uid) still works

**Risk/Notes**:
- Existing audit_logs without companyId will need migration during implementation

## Step 5 — Companies Rule Super_admin Fix

**Change**: Changed `/companies/{companyId}` write rule to `isCompanyAdminOfCompany(companyId)`

**Files**:
- `docs/rfc/authorization-rbac.md` (lines 708-711)

**Rationale**:
- ChatGPT identified: RFC says "super_admin system-wide" but rule used `belongsToCompany(companyId) && isCompanyAdmin()` which blocked super_admin from other tenants
- Fix: `isCompanyAdminOfCompany(companyId)` allows super_admin everywhere + company_admin only in their tenant

**Verification**:
- Manual review confirms consistency with RFC spec
- Added `|| isSuperAdminOnly()` to read rule for consistency

**Risk/Notes**:
- None - aligns with documented behavior

**Next**: Commit
