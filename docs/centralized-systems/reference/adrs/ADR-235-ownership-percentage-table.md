## 1. Context — Πρόβλημα

Ο εργολάβος πριν κατασκευάσει ένα οικοδομικό έργο, πρέπει να συντάξει τον **Πίνακα Ποσοστών** (Πίνακα Κατανομής Χιλιοστών Συνιδιοκτησίας). Αυτός ο πίνακας:

- Καθορίζει τα **χιλιοστά (‰)** κάθε ιδιοκτησίας επί του συνόλου του οικοπέδου
- Το σύνολο είναι **ΠΑΝΤΑ 1000‰**
- Καταχωρείται στο **Κτηματολόγιο** και στο **Υποθηκοφυλακείο**
- Αποτελεί αναπόσπαστο μέρος της **Πράξης Σύστασης Οριζόντιας Ιδιοκτησίας** (Ν. 3741/1929)
- Σε **αντιπαροχή**, κατανέμει τα χιλιοστά μεταξύ εργολάβου και οικοπεδούχου

### Τι λείπει σήμερα:

Η εφαρμογή υποστηρίζει `millesimalShares` ανά parking/storage (ADR-199), αλλά **δεν υπάρχει**:
1. Κεντρικός πίνακας κατανομής χιλιοστών ανά κτήριο
2. Μεθοδολογία υπολογισμού (εμβαδόν, αξία, κλπ.)
3. Σύνδεση χιλιοστών με κοινόχρηστα βάρη (ανελκυστήρας, θέρμανση)
4. Σενάριο αντιπαροχής εργολάβου–οικοπεδούχου
5. Validation ότι τα χιλιοστά ενός κτηρίου αθροίζουν πάντα σε 1000‰

## 3. Δομή Πίνακα Ποσοστών

### 3.1 Στήλες Πίνακα

| Στήλη | Περιγραφή | Τύπος | Παράδειγμα |
|-------|-----------|-------|------------|
| **Α/Α** | Αύξων αριθμός ιδιοκτησίας | number | 1, 2, 3... |
| **Κωδικός** | Κωδικός entity στο σύστημα | string | `A1-01`, `P-B1-03` |
| **Περιγραφή** | Τύπος + θέση ιδιοκτησίας | string | «Διαμέρισμα 1ου ορόφου» |
| **Κατηγορία** | Κύρια / Βοηθητική / Παρακολούθημα | enum | `main`, `auxiliary`, `appurtenance` |
| **Όροφος** | Θέση στο κτήριο | string | «1ος», «Υπόγειο», «Πιλοτή» |
| **Εμβαδόν (τ.μ.)** | Καθαρό εμβαδόν | number | 85.50 |
| **Χιλιοστά (‰)** | Μερίδιο επί οικοπέδου | number | 125 |
| **Ιδιοκτήτης** | Εργολάβος / Οικοπεδούχος / Αγοραστής | string | «Εργολάβος» |

### 3.2 Γραμμές Πίνακα

Ο πίνακας περιλαμβάνει **ΟΛΕΣ** τις αυτοτελείς ιδιοκτησίες **ΟΛΩΝ** των κτηρίων του οικοπέδου:

| Ομάδα | Τι περιλαμβάνει | Παράδειγμα |
|-------|-----------------|------------|
| **Κτήριο Α — Κύριες** | Διαμερίσματα, καταστήματα, γραφεία | Δ1 (1ος, 85τμ, 125‰) |
| **Κτήριο Α — Βοηθητικοί** | Κλειστές θέσεις, αποθήκες (με χιλιοστά) | Θ1 (Υπόγειο, 12τμ, 8‰) |
| **Υποσύνολο Κτηρίου Α** | | **420‰** |
| **Κτήριο Β — Κύριες** | ... | ... |
| **Υποσύνολο Κτηρίου Β** | | **350‰** |
| **Κτήριο Γ — Κύριες** | ... | ... |
| **Υποσύνολο Κτηρίου Γ** | | **230‰** |
| **ΓΕΝΙΚΟ ΣΥΝΟΛΟ** | Σύνολο ΟΛΩΝ των κτηρίων | **1000‰** |

### 3.2.1 Κάθετη Ιδιοκτησία — Ένας Πίνακας για Πολλά Κτήρια

> **ΚΑΝΟΝΑΣ**: Ο πίνακας χιλιοστών είναι **ΕΝΙΑΙΟΣ ΑΝΑ ΟΙΚΟΠΕΔΟ**, ΟΧΙ ανά κτήριο.
>
> Αν ένα οικόπεδο έχει 3 κτήρια (κάθετη ιδιοκτησία):
> - Τα κτήρια εμφανίζονται ως **ξεχωριστές ομάδες** (sections) στον πίνακα
> - Κάθε κτήριο δείχνει τις δικές του ιδιοκτησίες + **υποσύνολο χιλιοστών**
> - Στο τέλος, τα χιλιοστά **ΟΛΩΝ** των κτηρίων αθροίζουν σε **1000‰**
>
> **Παράδειγμα**: Οικόπεδο με 3 πολυκατοικίες:
> ```
> Κτήριο Α (5 δ/τα + 3 parking)  →  420‰
> Κτήριο Β (4 δ/τα + 2 parking)  →  350‰
> Κτήριο Γ (3 δ/τα + 1 κατ/μα)   →  230‰
>                          ΣΥΝΟΛΟ →  1000‰
> ```

### 3.3 Κοινόχρηστοι / Κοινόκτητοι Χώροι (ΔΕΝ έχουν χιλιοστά)

Αυτοί **δεν** εμφανίζονται στον πίνακα ως γραμμές αλλά αναφέρονται ξεχωριστά:

- Κλιμακοστάσια, ανελκυστήρας, κεντρική είσοδος
- Στέγη, ταράτσα (εκτός αν ανήκει ως αποκλειστική χρήση)
- Ανοιχτές θέσεις πιλοτής/ακάλυπτου (ΑΠ 23/2000)
- Λεβητοστάσιο, αποχετεύσεις, ηλεκτρολογικοί πίνακες

## 4. Μεθοδολογία Υπολογισμού Χιλιοστών

### 4.1 Τρεις Μέθοδοι

#### Μέθοδος Α: Κατ' Εμβαδόν (Αναλογική)

Η πιο απλή και συνηθέστερη. Τα χιλιοστά κατανέμονται **αναλογικά προς το εμβαδόν**.

```
χιλιοστά_i = (εμβαδόν_i / Σ(εμβαδών_όλων)) × 1000
```

**Πλεονεκτήματα**: Απλή, αντικειμενική, εύκολη εφαρμογή
**Μειονεκτήματα**: Δεν λαμβάνει υπόψη αξία (πεντοροφο vs ισόγειο κατάστημα)

**Παράδειγμα**:
| Ιδιοκτησία | Εμβαδόν | Υπολογισμός | Χιλιοστά |
|------------|---------|-------------|----------|
| Δ1 (1ος) | 85 τμ | 85/400 × 1000 | 213‰ |
| Δ2 (1ος) | 75 τμ | 75/400 × 1000 | 188‰ |
| Δ3 (2ος) | 90 τμ | 90/400 × 1000 | 225‰ |
| Δ4 (2ος) | 80 τμ | 80/400 × 1000 | 200‰ |
| Κατ1 (Ισόγειο) | 70 τμ | 70/400 × 1000 | 175‰ |
| **Σύνολο** | **400 τμ** | | **1001‰ → 1000‰** * |

> \* Στρογγυλοποίηση: Τα χιλιοστά στρογγυλοποιούνται σε ακέραιους αριθμούς. Η διαφορά (±1-2‰) κατανέμεται στη μεγαλύτερη ιδιοκτησία ή αναλογικά.

#### Μέθοδος Β: Κατ' Αντικειμενική Αξία (Σταθμισμένη)

Τα χιλιοστά κατανέμονται βάσει **αντικειμενικής αξίας**, που λαμβάνει υπόψη τον όροφο, τον προσανατολισμό, κλπ.

```
χιλιοστά_i = (αξία_i / Σ(αξιών_όλων)) × 1000

αξία_i = εμβαδόν_i × τιμή_ζώνης × Σ(συντελεστές_i)
```

**Συντελεστές** (βλ. §5):
- Συντελεστής ορόφου
- Συντελεστής αξίας (πρόσοψη, θέα, κλπ.)
- Συντελεστής εμπορικότητας (για καταστήματα)

**Πλεονεκτήματα**: Δίκαιη αποτίμηση, αντανακλά πραγματική αξία
**Μειονεκτήματα**: Πολυπλοκότερη, υποκειμενική εκτίμηση

#### Μέθοδος Γ: Κατ' Όγκον

Σπάνια χρησιμοποιείται. Τα χιλιοστά κατανέμονται βάσει **όγκου** (εμβαδόν × ύψος ορόφου).

```
χιλιοστά_i = (όγκος_i / Σ(όγκων_όλων)) × 1000

όγκος_i = εμβαδόν_i × ύψος_ορόφου_i
```

**Χρήση**: Βιομηχανικά κτήρια, αποθήκες με ημιώροφο, μεζονέτες

### 4.2 Στρογγυλοποίηση (Rounding Strategy)

Τα χιλιοστά πρέπει **ΠΑΝΤΑ** να αθροίζουν ακριβώς σε **1000‰**. Στρατηγική:

1. Υπολόγισε τα ακριβή (δεκαδικά) χιλιοστά
2. Στρογγυλοποίησε σε ακέραιους (floor)
3. Υπολόγισε τη **διαφορά** = 1000 - Σ(στρογγυλοποιημένων)
4. Κατάνειμε τη διαφορά στις ιδιοκτησίες με το **μεγαλύτερο δεκαδικό υπόλοιπο** (Largest Remainder Method)

> **Largest Remainder Method** (μέθοδος Hamilton): Εξασφαλίζει δίκαιη κατανομή στρογγυλοποίησης. Χρησιμοποιείται και σε εκλογικά συστήματα (αναλογική εκπροσώπηση).

## 5. Συντελεστές Κατανομής

### 5.1 Συντελεστής Ορόφου (ΣΟρ) — Επίσημος Πίνακας (ΠΟΛ 1149/1994, ΑΑΔΕ)

> **ΠΗΓΗ**: [ΠΟΛ 1149/1994 — ΑΑΔΕ](https://www.aade.gr/sites/default/files/2021-02/pol1149_1994.pdf)
> Ο συντελεστής ορόφου εξαρτάται από τον **Συντελεστή Εμπορικότητας (ΣΕ)** του δρόμου.

#### Πίνακας Α: ΣΕ < 1.5 (Κατοικίες — χαμηλή εμπορικότητα)

| Όροφος | ΣΟρ | Σημειώσεις |
|--------|-----|------------|
| Υπόγειο | 0.60 | Χαμηλότερη αξία |
| Ισόγειο | 0.90 | Κατοικία ισογείου |
| 1ος | 1.00 | **Βάση αναφοράς** |
| 2ος | 1.05 | +5% |
| 3ος | 1.10 | +10% |
| 4ος | 1.15 | +15% |
| 5ος+ | 1.20 | +20% |

#### Πίνακας Β: ΣΕ ≥ 1.5 και < 3 (Καταστήματα — υψηλή εμπορικότητα)

| Όροφος | ΣΟρ | Σημειώσεις |
|--------|-----|------------|
| Υπόγειο | 0.60 | Χαμηλότερη αξία |
| Ισόγειο | **1.20** | Υψηλή αξία — εμπορική χρήση |
| 1ος | 1.10 | Πάνω από ισόγειο |
| 2ος | 1.05 | Μείωση εμπορικής αξίας |
| 3ος | 1.10 | Αύξηση λόγω θέας |
| 4ος | 1.15 | |
| 5ος | 1.20 | |
| 6ος+ | 1.25 | |

> **ΚΑΝΟΝΑΣ — DEFAULTS + OVERRIDE**:
> - Οι συντελεστές ορόφου είναι **προ-συμπληρωμένοι** βάσει ΠΟΛ 1149/1994
> - Το σύστημα επιλέγει αυτόματα Πίνακα Α ή Β βάσει του ΣΕ που εισάγει ο μηχανικός
> - Ο μηχανικός **βλέπει** τους defaults και μπορεί να τους **τροποποιήσει** αν χρειαστεί
> - Αν τροποποιηθεί, εμφανίζεται ένδειξη ότι η τιμή διαφέρει από την επίσημη
>
> Ο ΣΕ (Συντελεστής Εμπορικότητας) ορίζεται ανά δρόμο στα φύλλα του valuemaps.gov.gr.
> Ο μηχανικός εισάγει τον ΣΕ μαζί με την τιμή ζώνης.

### 5.2 Λοιποί Συντελεστές Αντικειμενικής Αξίας (ΠΟΛ 1149/1994)

Πέραν του ΣΟρ, η αντικειμενική αξία περιλαμβάνει:

| Συντελεστής | Κωδικός | Τι επηρεάζει | Εισαγωγή |
|-------------|---------|--------------|----------|
| Εμπορικότητας | ΣΕ | Εμπορικότητα δρόμου | Χειροκίνητα (από valuemaps) |
| Παλαιότητας | ΣΠ | Ηλικία κτηρίου | Αυτόματα (βάσει έτους κατασκευής) |
| Πρόσοψης | — | Κύρια/δευτερεύουσα πρόσοψη | Χειροκίνητα |
| Ορόφου | ΣΟρ | Βλ. πίνακες Α/Β παραπάνω | Αυτόματα (βάσει ορόφου + ΣΕ) |

### 5.3 Κοινόχρηστα Βάρη — ΞΕΧΩΡΙΣΤΟΣ ΠΙΝΑΚΑΣ

> **ΑΠΟΦΑΣΗ**: Τα κοινόχρηστα βάρη (ανελκυστήρας, θέρμανση, γενικά κοινόχρηστα) είναι **ΞΕΧΩΡΙΣΤΟΣ ΠΙΝΑΚΑΣ** από τον πίνακα χιλιοστών. Δεν αναμιγνύονται στο ίδιο document.
>
> - **Πίνακας Χιλιοστών (ADR-235)**: Χιλιοστά οικοπέδου, αντιπαροχή, ιδιοκτήτες
> - **Πίνακας Κοινοχρήστων (μελλοντικό ADR)**: % ανελκυστήρα, % θέρμανσης, % γενικών κοινοχρήστων
>
> Ο πίνακας κοινοχρήστων **βασίζεται** στα χιλιοστά ως σημείο εκκίνησης, αλλά είναι ανεξάρτητο έγγραφο.
> Η τεκμηρίωση και υλοποίησή του θα γίνει σε ξεχωριστό ADR.

> **ΚΡΙΣΙΜΗ ΔΙΕΥΚΡΙΝΙΣΗ**: Χιλιοστά λαμβάνουν **ΜΟΝΟ**:
> 1. **Διαμερίσματα / Καταστήματα / Γραφεία** (κύριες ιδιοκτησίες)
> 2. **Κλειστά πάρκινγκ** (γκαράζ — κύριοι χώροι με τοίχους + πόρτα)

| Θέση στάθμευσης ακάλυπτου | `parking_spot` | ❌ ΟΧΙ | ❌ ΟΧΙ | ΑΠ 23/2000 |
| Υπαίθρια θέση στάθμευσης | `parking_spot` | ❌ ΟΧΙ | ❌ ΟΧΙ | Κοινόχρηστος χώρος |

## 7. Σενάριο Αντιπαροχής

### 7.1 Τι είναι η Αντιπαροχή

Στην αντιπαροχή, ο/οι **οικοπεδούχος/-οι** παραχωρούν οικόπεδο στον **εργολάβο** σε αντάλλαγμα ιδιοκτησιών στο νέο κτήριο. Ο πίνακας χιλιοστών κατανέμεται:

```
1000‰ = χιλιοστά_εργολάβου + Σ(χιλιοστά_οικοπεδούχων)
```

> **ΚΑΝΟΝΑΣ — ΠΟΛΛΑΠΛΟΙ ΟΙΚΟΠΕΔΟΥΧΟΙ**: Μπορεί να υπάρχουν **πολλοί οικοπεδούχοι** (π.χ. κληρονόμοι, συνιδιοκτήτες). Τα χιλιοστά που αναλογούν στους οικοπεδούχους **μοιράζονται μεταξύ τους βάσει του ποσοστού αντιπαροχής** που έχει ο καθένας.
>
> **Παράδειγμα**: Αντιπαροχή 40%, 3 αδέρφια με ίσα μερίδια (1/3 έκαστος):
> - Εργολάβος: 600‰
> - Οικοπεδούχος Α: 400‰ × 1/3 = 133‰
> - Οικοπεδούχος Β: 400‰ × 1/3 = 133‰
> - Οικοπεδούχος Γ: 400‰ × 1/3 = 134‰ (στρογγυλοποίηση)
> - **Σύνολο: 1000‰**

### 7.2 Τυπική Κατανομή

| Ποσοστό Αντιπαροχής | Εργολάβος | Οικοπεδούχος | Σημειώσεις |
|---------------------|-----------|--------------|------------|
| **30–35%** (χαμηλή) | 650–700‰ | 300–350‰ | Περιοχές χαμηλής ζήτησης |
| **40–45%** (μέση) | 550–600‰ | 400–450‰ | Μέσος όρος Αθήνας/Θεσσαλονίκης |
| **50–55%** (υψηλή) | 450–500‰ | 500–550‰ | Premium περιοχές (Κολωνάκι, Γλυφάδα) |
| **60%+** (πολύ υψηλή) | 400‰ | 600‰ | Σπάνια — πολυτελείς περιοχές |

### 7.4 Κανόνες Αντιπαροχής

1. Η κατανομή συμφωνείται **ΠΡΟ** της κατασκευής
2. Καταγράφεται σε **συμβολαιογραφική πράξη**
3. Ο οικοπεδούχος μεταβιβάζει οικόπεδο στον εργολάβο κατά τα χιλιοστά του
4. Μετά την οικοδομή, κάθε πλευρά λαμβάνει τα συμφωνημένα ακίνητα
5. Η αντιπαροχή αφορά **μόνο κύριες + βοηθητικές** ιδιοκτησίες (με χιλιοστά), **ΟΧΙ** κοινόχρηστα

## 8. Σχέση με Υπάρχοντα ADRs

### 8.1 ADR-199 — Sales Appurtenances

| Σημείο σύνδεσης | ADR-199 | ADR-235 (αυτό) |
|-----------------|---------|-----------------|
| `millesimalShares` field | ✅ Ήδη στο ParkingSpot/StorageUnit | Θα χρησιμοποιηθεί ως source data |
| `canSellIndependently()` | ✅ Helper function | Καθορίζει ποια entities μπαίνουν στον πίνακα |
| Κατηγοριοποίηση (2.2) | ✅ Νομική ανάλυση | Βάση για κατηγοριοποίηση γραμμών |
| Πώληση παρακολουθημάτων | ✅ Transaction chain | Μετά τον πίνακα — πώληση βάσει χιλιοστών |

### 8.2 ADR-234 — Payment Plans

| Σημείο σύνδεσης | ADR-234 | ADR-235 (αυτό) |
|-----------------|---------|-----------------|
| Τιμή ακινήτου | ✅ `totalAmount` | Τιμή μπορεί να βασίζεται σε χιλιοστά × αξία/‰ |
| Εργολάβος/Οικοπεδούχος | ❌ Δεν διακρίνει | ✅ Ξεχωρίζει ιδιοκτήτη ανά γραμμή |

### 8.3 ADR-182 — Parking/Storage Collections

Τα Firestore collections `parking_spots` και `storage_units` ήδη υποστηρίζουν `millesimalShares`. Ο πίνακας χιλιοστών θα **συγκεντρώνει** δεδομένα από:
- `units` collection (ΠΑΝΤΑ με χιλιοστά)
- `parking_spots` collection (ΜΟΝΟ αν `millesimalShares > 0`)
- `storage_units` collection (ΜΟΝΟ αν `millesimalShares > 0`)

## 9. Proposed Data Model (TypeScript Interfaces — Χωρίς Υλοποίηση)

> **ΣΗΜΑΝΤΙΚΟ**: Τα παρακάτω interfaces είναι **μόνο πρόταση**. Η υλοποίηση θα γίνει σε μελλοντική φάση μετά από έγκριση.

```typescript
// ============================================================
// ADR-235: Ownership Percentage Table — Proposed Interfaces
// ============================================================

/**
 * Μεθοδολογία υπολογισμού χιλιοστών
 */
type CalculationMethod = 'area' | 'value' | 'volume';

/**
 * Κατηγορία ιδιοκτησίας στον πίνακα
 */

/**
 * Ιδιοκτήτης στο σενάριο αντιπαροχής
 */
type OwnerParty = 'contractor' | 'landowner' | 'buyer' | 'unassigned';

/**
 * Οικοπεδούχος στο σενάριο αντιπαροχής (μπορεί να υπάρχουν πολλοί)

  /** Contact ID του οικοπεδούχου */
  readonly contactId: string;
  /** Ονοματεπώνυμο */
  readonly name: string;

  readonly landOwnershipPct: number;
  /** Χιλιοστά που αναλογούν (υπολογίζεται αυτόματα) */
  readonly allocatedShares: number;
}

/**
 * Μία γραμμή στον πίνακα ποσοστών — αντιπροσωπεύει μία αυτοτελή ιδιοκτησία
 */
interface OwnershipTableRow {
  /** Αύξων αριθμός */
  readonly ordinal: number;
  /** Reference στο κτήριο (για ομαδοποίηση στον πίνακα) */
  readonly buildingId: string;
  /** Reference στο entity (unit/parking/storage) */
  readonly entityRef: {
    readonly collection: 'units' | 'parking_spots' | 'storage_units';
    readonly id: string;
  };
  /** Κωδικός entity (π.χ. "A1-01", "P-B1-03") */
  readonly entityCode: string;
  /** Περιγραφή */
  readonly description: string;
  /** Κατηγορία ιδιοκτησίας */
  readonly category: PropertyCategory;
  /** Όροφος */
  readonly floor: string;
  /** Καθαρό εμβαδόν (τ.μ.) */
  readonly areaSqm: number;
  /** Χιλιοστά οικοπέδου (‰) — ΑΚΕΡΑΙΟΣ */
  readonly millesimalShares: number;
  // Κοινόχρηστα βάρη (%, ανελκυστήρας, θέρμανση) → ΞΕΧΩΡΙΣΤΟΣ ΠΙΝΑΚΑΣ (μελλοντικό ADR)
  /** Ιδιοκτήτης (αντιπαροχή) */
  readonly ownerParty: OwnerParty;
  /** Contact ID αγοραστή (αν πωλήθηκε) */
  readonly buyerContactId: string | null;
}

/**
 * Συντελεστές υπολογισμού (Μέθοδος Β: Αντικειμενική Αξία)
 */
interface CalculationCoefficients {
  /** Συντελεστής ορόφου */
  readonly floorCoefficient: number;
  /** Συντελεστής αξίας (πρόσοψη, θέα, φωτισμός) */
  readonly valueCoefficient: number;
  /** Συντελεστής εμπορικότητας (μόνο για καταστήματα) */
  readonly commercialCoefficient: number | null;
}

/**
 * Σύνοψη αντιπαροχής
 */
interface BartexSummary {
  /** Ποσοστό αντιπαροχής (%) — π.χ. 40 σημαίνει 40% στους οικοπεδούχους */
  readonly bartexPercentage: number;
  /** Σύνολο χιλιοστών εργολάβου */
  readonly contractorShares: number;
  /** Σύνολο χιλιοστών ΟΛΩΝ των οικοπεδούχων */
  readonly totalLandownerShares: number;
  /** Πλήθος ιδιοκτησιών εργολάβου */
  readonly contractorPropertyCount: number;
  /** Πλήθος ιδιοκτησιών οικοπεδούχων (συνολικά) */
  readonly landownerPropertyCount: number;
  /** Λίστα οικοπεδούχων — μπορεί να είναι πολλοί (κληρονόμοι, συνιδιοκτήτες) */
  readonly landowners: ReadonlyArray<LandownerEntry>;
}

/**
 * Πλήρης Πίνακας Ποσοστών Συνιδιοκτησίας — ένας ανά κτήριο
 */
interface OwnershipPercentageTable {
  /** Firestore document ID */
  readonly id: string;
  /** Reference στο project (= οικόπεδο) — ΕΝΑΣ πίνακας ανά οικόπεδο */
  readonly projectId: string;
  /** Κτήρια που συμμετέχουν (κάθετη ιδιοκτησία = πολλά) */
  readonly buildingIds: ReadonlyArray<string>;
  /** Ημερομηνία σύνταξης */
  readonly createdAt: FirebaseFirestore.Timestamp;
  /** Τελευταία ενημέρωση */
  readonly updatedAt: FirebaseFirestore.Timestamp;
  /** Τιμή ζώνης (€/τ.μ.) — εισάγεται χειροκίνητα από τον μηχανικό */
  readonly zonePrice: number;
  /** Μεθοδολογία υπολογισμού */
  readonly calculationMethod: CalculationMethod;
  /** Γραμμές πίνακα — ΟΛΕΣ οι αυτοτελείς ιδιοκτησίες */
  readonly rows: ReadonlyArray<OwnershipTableRow>;
  /** Σύνολο χιλιοστών — ΠΡΕΠΕΙ = 1000 */
  readonly totalShares: 1000;
  /** Σύνοψη ανά κατηγορία */
  readonly summaryByCategory: {
    readonly main: { readonly count: number; readonly shares: number };
    readonly auxiliary: { readonly count: number; readonly shares: number };
  };
  /** Σενάριο αντιπαροχής (null αν δεν ισχύει) */
  readonly bartex: BartexSummary | null;
  /** Σημειώσεις (ελεύθερο κείμενο) */
  readonly notes: string | null;
  /** Αρ. πράξης σύστασης (notarial deed) */

  /** Συμβολαιογράφος */
  readonly notary: string | null;
  /** Κωδικοί ΚΑΕΚ (αν έχει γίνει κτηματογράφηση) */
  readonly kaekCodes: ReadonlyArray<string> | null;
  /** Status */
  readonly status: 'draft' | 'finalized' | 'registered';
  /** Τρέχουσα έκδοση */
  readonly version: number;
}

/**
 * Snapshot μιας παλαιότερης έκδοσης του πίνακα (revision history)
 * Αποθηκεύεται ως subcollection: ownership_tables/{tableId}/revisions/{revisionId}

interface OwnershipTableRevision {

  readonly version: number;
  /** Πλήρες snapshot του πίνακα τη στιγμή του finalize */
  readonly snapshot: Omit<OwnershipPercentageTable, 'id' | 'version'>;
  /** Ποιος έκανε finalize αυτή την έκδοση */
  readonly finalizedBy: string;
  /** Πότε έγινε finalize */
  readonly finalizedAt: FirebaseFirestore.Timestamp;
  /** Λόγος αλλαγής (αν ξεκλειδώθηκε και δημιουργήθηκε νέα έκδοση) */
  readonly changeReason: string | null;
}

### 10.3 Ροή Εργασίας (Workflow)

> **ΚΑΝΟΝΑΣ**: Ο πίνακας ποσοστών καθορίζεται από τον **μηχανικό**, ΟΧΙ τον συμβολαιογράφο.
> Ο συμβολαιογράφος τον ενσωματώνει στην Πράξη Σύστασης, αλλά η σύνταξη είναι αρμοδιότητα του μηχανικού.

#### Βήμα 0: Εισαγωγή Τιμής Ζώνης (Manual Input)
Ο μηχανικός εισάγει χειροκίνητα την **τιμή ζώνης (€/τ.μ.)** που βρίσκει από:
- [valuemaps.gov.gr](https://valuemaps.gov.gr/) — Επίσημος χάρτης Υπ. Οικονομικών
- [ΑΑΔΕ — Φύλλα Υπολογισμού](https://www.aade.gr/en/property-value-calculation-sheets)

> **ΚΑΝΟΝΑΣ — ΜΙΑ ΤΙΜΗ ΖΩΝΗΣ ΑΝΑ ΟΙΚΟΠΕΔΟ**: Αν το οικόπεδο βλέπει σε δρόμους **διαφορετικών ζωνών** (γωνιακό, κλπ.), σύμφωνα με τη νομοθεσία λαμβάνεται πάντα η **ΜΕΓΑΛΥΤΕΡΗ** τιμή ζώνης.
> Αυτό ισχύει τόσο για κυκλικές ζώνες όσο και για γραμμικές ζώνες (ΠΟΛ. 1149/1994).
> Πηγή: [Υπ. Οικονομικών — FAQ](https://minfin.gov.gr/oikonomiki-politiki/sychnes-erotiseis/)

> **ΣΗΜΕΙΩΣΗ — Απουσία API**: Δεν υπάρχει δημόσιο API για αντικειμενικές αξίες στην Ελλάδα (2026).
> Η ΑΑΔΕ/Υπ. Οικονομικών παρέχει μόνο web UI (valuemaps.gov.gr), όχι programmatic access.
> Η μοναδική εμπορική λύση είναι το [eaxies.gr](https://www.eaxies.gr/) (B-logica) — web services επί πληρωμή.
> **Μελλοντικά**: Αν ανοίξει δημόσιο API ή γίνει συνδρομή σε eaxies.gr, θα ενσωματωθεί αυτόματα.

#### Βήμα 1: Αυτόματη Σύνταξη (Auto-Calculate)
Το σύστημα **υπολογίζει αυτόματα** ολόκληρο τον πίνακα βάσει:
- Τιμής ζώνης (€/τ.μ.) που εισήγαγε ο μηχανικός
- Νομοθεσίας (Ν. 3741/1929, ΝΟΚ)
- Επίσημων κανόνων φορέων (ΤΕΕ, Κτηματολόγιο)
- Μεθοδολογίας υπολογισμού (εμβαδόν/αξία/όγκος)
- Δεδομένων κτηρίου (units, parking, storage + εμβαδά + όροφοι)

#### Βήμα 2: Παρέμβαση Μηχανικού (Manual Override)
Ο μηχανικός **μπορεί να τροποποιήσει** οποιαδήποτε τιμή:
- Αλλαγή χιλιοστών σε συγκεκριμένη ιδιοκτησία
- Προσαρμογή συντελεστών (ορόφου, αξίας, κλπ.)
- Αλλαγή κατανομής κοινοχρήστων/ανελκυστήρα/θέρμανσης
- Το σύστημα δείχνει real-time αν το σύνολο = 1000‰ ή όχι

#### Βήμα 3: Κλείδωμα (Lock/Finalize)
Όταν ο μηχανικός είναι ικανοποιημένος:
- **Finalize** → Ο πίνακας κλειδώνει, δεν μπορεί να αλλάξει
- Μόνο ο μηχανικός (ή admin) μπορεί να ξεκλειδώσει
- Ο κλειδωμένος πίνακας εξάγεται σε PDF για τον συμβολαιογράφο

#### Βήμα 4: Revision History (Ιστορικό Εκδόσεων)
Αν ο μηχανικός ξεκλειδώσει έναν finalized πίνακα:
- Η **προηγούμενη έκδοση** αποθηκεύεται αυτόματα ως snapshot (revision)
- Κάθε revision έχει: αριθμό έκδοσης, ημερομηνία, ποιος την έκανε, λόγο αλλαγής
- Ο μηχανικός μπορεί να δει και να συγκρίνει παλιές εκδόσεις
- **Δεν διαγράφεται ποτέ** revision — πλήρες audit trail

| Λειτουργία | Περιγραφή |
|-----------|-----------|
| **Auto-populate** | Κουμπί «Γέμισε από κτήριο» — φέρνει units/parking/storage με χιλιοστά |

### 10.4 Validation Rules

1. **Σύνολο χιλιοστών = 1000‰** — ΠΑΝΤΑ (μη διαπραγματεύσιμο)
2. **Κάθε γραμμή ≥ 1‰** — Δεν υπάρχει ιδιοκτησία με 0 χιλιοστά στον πίνακα
3. **Κάθε γραμμή αντιστοιχεί σε entity** — Δεν υπάρχουν «φανταστικές» ιδιοκτησίες
4. **Ένας πίνακας ανά οικόπεδο/project** — Αν υπάρχουν πολλά κτήρια (κάθετη ιδιοκτησία), εμφανίζονται ως ξεχωριστές ομάδες στον ΙΔΙΟ πίνακα
5. **Finalized πίνακας = immutable** — Μετά τη μεταγραφή, δεν αλλάζει

## 10.5 Document ID Generation (Enterprise SSoT — ADR-017)

**Πηγή αλήθειας**: `enterprise-id.service.ts` — deterministic composite key pattern.

| Entity | Generator | Pattern | Παράδειγμα |
|--------|-----------|---------|------------|
| Ownership Table | `generateOwnershipTableId(projectId)` | `owntbl_{projectId}` | `owntbl_proj_a1b2c3d4-...` |
| Revision | `generateOwnershipRevisionId(version)` | `owntbl_rev_v{N}` | `owntbl_rev_v3` |

**Prefix**: `owntbl` (registered in `enterprise-id-prefixes.ts`)

**Γιατί deterministic**: Σχέση 1:1 table↔project. Deterministic key = O(1) direct document access χωρίς Firestore query. Google Bigtable composite key pattern.

> **ΚΑΝΟΝΑΣ**: ΑΠΑΓΟΡΕΥΕΤΑΙ inline ID generation (`ownership_{projectId}`). Μόνο μέσω `generateOwnershipTableId()`.

              └── changeReason: string | null
```

> **Κανόνας**: Ένα document ανά **οικόπεδο/project**. Αν το οικόπεδο έχει πολλά κτήρια (κάθετη ιδιοκτησία), όλα περιλαμβάνονται στον ίδιο πίνακα, ομαδοποιημένα ανά `buildingId`. Αν χρειάζεται revision history, νέο document με `parentTableId` reference.
