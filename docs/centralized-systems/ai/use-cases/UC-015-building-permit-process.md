# UC-015: Αδειοδότηση (Building Permit Process)

> **Parent ADR**: [ADR-169 - Modular AI Architecture](../../reference/adrs/ADR-169-modular-ai-architecture.md)
> **Pipeline**: [pipeline.md](../pipeline.md)
> **Σχετικά UCs**: [UC-014](./UC-014-property-sale-process.md) (πώληση — χρειάζεται άδεια), [UC-016](./UC-016-procurement-process.md) (προμήθεια — ξεκινάει μετά την άδεια)

## Trigger

Νέο έργο ξεκινάει ή υπάρχον έργο χρειάζεται αδειοδότηση (νέα άδεια, αναθεώρηση, αλλαγή χρήσης). Η AI παρακολουθεί ολόκληρη τη διαδικασία μέχρι την έκδοση.

---

## Στάδια Διαδικασίας

### Επισκόπηση

```
ΜΕΛΕΤΕΣ → ΕΓΚΡΙΣΕΙΣ ΦΟΡΕΩΝ → ΥΠΟΒΟΛΗ e-ΑΔΕΙΕΣ → ΕΚΔΟΣΗ → ΑΝΑΓΓΕΛΙΑ ΕΝΑΡΞΗΣ → ΕΠΙΘΕΩΡΗΣΕΙΣ → ΠΕΡΑΙΩΣΗ
```

---

### Στάδιο 1: Μελέτες

| Πεδίο | Λεπτομέρεια |
|-------|-------------|
| **Κατάσταση** | `studies_in_progress` → `studies_complete` |
| **Ενέργειες AI** | Tracking ανάθεσης και παράδοσης κάθε μελέτης, alerts καθυστέρησης |

**Checklist μελετών**:

| Μελέτη | Αρμόδιος | Εξάρτηση | SLA |
|--------|----------|----------|-----|
| ☐ Τοπογραφικό | Τοπογράφος | — | ~10 μέρες |
| ☐ Αρχιτεκτονική μελέτη | `architect` | Τοπογραφικό | ~30 μέρες |
| ☐ Στατική μελέτη | Πολιτικός μηχανικός | Αρχιτεκτονική | ~20 μέρες |
| ☐ Η/Μ μελέτη (Ηλεκτρομηχανολογικά) | `mechanicalEngineer` | Αρχιτεκτονική | ~20 μέρες |
| ☐ Ενεργειακή μελέτη (ΚΕΝΑΚ) | Ενεργειακός μελετητής | Αρχιτεκτονική + Η/Μ | ~15 μέρες |
| ☐ Μελέτη πυροπροστασίας | `mechanicalEngineer` | Αρχιτεκτονική | ~10 μέρες |
| ☐ Μελέτη προσβασιμότητας (ΑμεΑ) | `architect` | Αρχιτεκτονική | ~5 μέρες |

**Εξαρτήσεις**: Η AI γνωρίζει τη σειρά — δεν μπορεί να ξεκινήσει στατική χωρίς αρχιτεκτονική, δεν μπορεί να ξεκινήσει ενεργειακή χωρίς Η/Μ.

**Παράλληλες εργασίες**: Στατική + Η/Μ + Πυροπροστασία μπορούν παράλληλα (αρκεί να υπάρχει αρχιτεκτονική).

**Alerts**:
- Μελέτη >SLA+10 μέρες → ειδοποίηση `architect` + μελετητή
- Μελέτη >SLA+20 μέρες → κλιμάκωση σε `owner`

---

### Στάδιο 2: Εγκρίσεις Φορέων

| Πεδίο | Λεπτομέρεια |
|-------|-------------|
| **Κατάσταση** | `approvals_pending` → `approvals_complete` |
| **Ενέργειες AI** | Tracking υποβολών, monitoring αποτελεσμάτων, alerts για deadlines |

**Πιθανοί φορείς** (ανάλογα το έργο):

| Φορέας | Πότε χρειάζεται | SLA (κατά μ.ό.) | Αρμόδιος |
|--------|----------------|-----------------|----------|
| ☐ Αρχαιολογική Υπηρεσία | Πάντα (εντός σχεδίου) | ~30-60 μέρες | `architect` |
| ☐ Δασαρχείο | Αν κοντά σε δάσος/ρέμα | ~30-90 μέρες | `architect` |
| ☐ Πυροσβεστική | Κτίρια >3 ορόφων, εμπορικά | ~15-30 μέρες | `mechanicalEngineer` |
| ☐ ΔΕΗ/ΔΕΔΔΗΕ | Σύνδεση ρεύματος | ~15 μέρες | `mechanicalEngineer` |
| ☐ ΕΥΔΑΠ/ΔΕΥΑ | Σύνδεση ύδρευσης/αποχέτευσης | ~15 μέρες | `mechanicalEngineer` |
| ☐ Υπηρεσία Δόμησης (ΥΔΟΜ) | Πάντα | ~30-60 μέρες | `architect` |
| ☐ Συμβούλιο Αρχιτεκτονικής | Αν παραδοσιακός οικισμός | ~30-60 μέρες | `architect` |

**ΚΑΝΟΝΑΣ**: Η AI δεν επιτρέπει υποβολή στο e-Άδειες αν δεν έχουν ολοκληρωθεί οι απαιτούμενες εγκρίσεις.

**Alerts**:
- Φορέας >SLA χωρίς απάντηση → "Η Αρχαιολογική δεν έχει απαντήσει εδώ και 45 μέρες — να κάνουμε follow-up;"
- Απόρριψη → ειδοποίηση `architect` + `owner`, λόγος απόρριψης, αναμονή διορθωτικής υποβολής

---

### Στάδιο 3: Υποβολή στο e-Άδειες

| Πεδίο | Λεπτομέρεια |
|-------|-------------|
| **Κατάσταση** | `permit_submitted` → `permit_under_review` |
| **Ενέργειες AI** | Ειδοποίηση για υποβολή, tracking ημερομηνίας, monitoring κατάστασης |

**Checklist υποβολής**:

| Έγγραφο | Status |
|---------|--------|
| ☐ Όλες οι μελέτες (Στάδιο 1) | Πρέπει ✅ |
| ☐ Όλες οι εγκρίσεις φορέων (Στάδιο 2) | Πρέπει ✅ |
| ☐ Αμοιβές μηχανικών (e-Αμοιβές) | Πρέπει ✅ |
| ☐ Εισφορές (ΔΕΗ, ΕΥΔΑΠ, Δήμος) | Πρέπει ✅ |
| ☐ ΦΕΜ (Φύλλο Ελέγχου Μελέτης) | Αν απαιτείται |

### Στάδιο 4: Έκδοση Αδείας

| Πεδίο | Λεπτομέρεια |
|-------|-------------|
| **Κατάσταση** | `permit_issued` |
| **Ενέργειες AI** | Καταγραφή αριθμού αδείας, ημερομηνίας λήξης, ειδοποίηση όλων |

**Μετά την έκδοση**:
- Αποθήκευση αδείας στο project
- Ενημέρωση project status
- Ειδοποίηση: `owner`, `architect`, `siteManager`, `civilEngineer`
- Καταγραφή ημερομηνίας λήξης (συνήθως 4 χρόνια)

**Alert λήξης**: 6 μήνες πριν τη λήξη → "Η οικοδομική άδεια λήγει στις [ημερομηνία] — χρειάζεται αναθεώρηση;"

---

### Στάδιο 5: Αναγγελία Έναρξης Εργασιών

| Πεδίο | Λεπτομέρεια |
|-------|-------------|
| **Κατάσταση** | `construction_start_notified` |
| **Ενέργειες AI** | Υπενθύμιση υποβολής, tracking |

**Checklist**:

| Ενέργεια | Αρμόδιος |
|----------|----------|
| ☐ Αναγγελία στο e-Άδειες | `architect` |
| ☐ Ανάρτηση πινακίδας εργοταξίου | `siteManager` |
| ☐ Βιβλίο ημερήσιου ελέγχου | `siteManager` |
| ☐ Σχέδιο ασφάλειας (ΣΑΥ/ΦΑΥ) | `civilEngineer` |

**ΚΑΝΟΝΑΣ**: Δεν ξεκινάει κατασκευή χωρίς αναγγελία. Η AI μπλοκάρει transition σε construction phase αν λείπει.

---

### Στάδιο 6: Επιθεωρήσεις κατά την κατασκευή

| Πεδίο | Λεπτομέρεια |
|-------|-------------|
| **Κατάσταση** | `inspections_in_progress` |
| **Ενέργειες AI** | Υπενθύμιση ελεγκτή δόμησης ανά φάση, tracking αποτελεσμάτων |

**Τυπικές επιθεωρήσεις**:

| Φάση | Επιθεώρηση | Ελεγκτής |
|------|-----------|----------|
| ☐ Εκσκαφή/Θεμελίωση | Έλεγχος εδάφους, θεμελίων | Ελεγκτής δόμησης |
| ☐ Σκελετός | Έλεγχος σκυροδέματος, οπλισμού | Ελεγκτής δόμησης |
| ☐ Τοιχοποιία | Έλεγχος τοιχοδομής | Ελεγκτής δόμησης |
| ☐ Η/Μ εγκαταστάσεις | Ηλεκτρολογικά, υδραυλικά | Ελεγκτής δόμησης |
| ☐ Ολοκλήρωση | Τελικός έλεγχος | Ελεγκτής δόμησης |

**Alert**: Αν φάση κατασκευής ολοκληρώθηκε χωρίς επιθεώρηση → "Η θεμελίωση ολοκληρώθηκε αλλά ο ελεγκτής δεν έχει κληθεί"

---

### Στάδιο 7: Βεβαίωση Περαίωσης

| Πεδίο | Λεπτομέρεια |
|-------|-------------|
| **Κατάσταση** | `completion_pending` → `completion_certified` |
| **Ενέργειες AI** | Checklist τελικών εγγράφων, υποβολή, tracking |

**Checklist περαίωσης**:

| Έγγραφο | Αρμόδιος |
|---------|----------|
| ☐ Τελική επιθεώρηση ελεγκτή δόμησης | Ελεγκτής |
| ☐ Πιστοποιητικό ενεργειακής απόδοσης (ΠΕΑ) | Ενεργειακός επιθεωρητής |
| ☐ Βεβαίωση ηλεκτρολόγου | `mechanicalEngineer` |
| ☐ Βεβαίωση υδραυλικού | `mechanicalEngineer` |
| ☐ Πιστοποιητικό πυροπροστασίας | Πυροσβεστική (αν απαιτείται) |
| ☐ Δήλωση περαίωσης στο e-Άδειες | `architect` |
| ☐ Σύνδεση ρεύματος (οριστική) | `mechanicalEngineer` |
| ☐ Αριθμός παροχής ρεύματος/νερού | `mechanicalEngineer` |

**Μετά την περαίωση**:
- Project status → `completed`
- Ενεργοποίηση εγγυήσεων ανά unit
- Ξεκλείδωμα πωλήσεων σε units χωρίς αγοραστή
- Ειδοποίηση `salesManager` → "Το έργο ολοκληρώθηκε — τα εξής units είναι διαθέσιμα: [λίστα]"

## Κανόνες

- **Εξαρτήσεις είναι αμετάκλητες**: Η AI δεν επιτρέπει παράληψη βήματος — κάθε στάδιο απαιτεί ολοκλήρωση του προηγούμενου
- **Ημερομηνίες λήξης**: Κάθε έγγραφο/έγκριση με λήξη → auto-alert 30 μέρες πριν
- **Αλλαγές σε μελέτες**: Αν αλλάξει αρχιτεκτονική μετά την υποβολή → AI σημαίνει: "Χρειάζεται αναθεώρηση στατικής/Η-Μ/ενεργειακής"
- **Αναθεώρηση αδείας**: Αν αλλαχτεί ο σχεδιασμός → νέα υποβολή, η AI ξεκινάει νέο κύκλο tracking
- **Νομική συμμόρφωση**: Η AI δεν αντικαθιστά τον μηχανικό — ενημερώνει, υπενθυμίζει, αλλά η ευθύνη παραμένει στον αρμόδιο μηχανικό
